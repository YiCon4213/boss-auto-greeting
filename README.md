# boss-auto-greeting
基于篡改猴/油猴的boss直聘自动打招呼脚本

## 本地简历投递 Agent 开发状态

项目正在保留油猴独立模式的前提下，增加本机 FastAPI + SQLite Agent。Phase 3“分析、话术与审批工作台”的实现与退出门禁已完成：

- 本地服务默认绑定 `127.0.0.1:8765`，Alembic head 为 `0003`。
- 浏览器任务支持白名单类型、30 秒租约、worker 所有权、进度序号和幂等终态。
- `python -m agent_app.cli show-browser-token` 只在本机终端显式显示浏览器令牌，HTTP API 不回显。
- 油猴新增默认关闭的 `AgentBridge` 和只采集不发送的 `BatchCollector`；独立模式入口保持原实现。
- 强岗位 ID、JD 指纹、不可变快照和重复审计已经落地。
- OpenAI-compatible 模型适配、可关闭双评分、事实可追溯问候语和安全失败记录已经落地。
- 本地双栏工作台支持冻结快照审阅、选择、终稿编辑和人工批准；批准生成不可变队列，但不触发发送。
- 81 项自动化测试通过，`agent_app` 覆盖率 92.42%，工作台与 Userscript 语法检查通过。

2026-07-22 已按 [两岗位人工验证清单](docs/manual-testing/phase-2-collection.md) 在用户本人 Chrome 完成只读采集：全过程无聊天、无发送，独立模式回归通过。Phase 3 的 [工作台验证记录](docs/manual-testing/phase-3-workbench.md) 也已通过：桌面双栏正常，390×844 窄屏无横向溢出且批准按钮完整可见；验证没有访问 BOSS、调用真实模型或执行发送。

安全发送、自然语言命令和外部子 Agent 契约尚未实现，`ApprovedQueueRunner` 仍为占位，当前代码仍不能用于 Agent 模式投递。开发和安全约束请先阅读 [AGENTS.md](AGENTS.md)，后续阶段入口见 [docs/README.md](docs/README.md)。

本地开发验证：

```powershell
.\.venv\Scripts\python.exe -m pytest -q
node --check .\zhipin-auto-greeting.user.js
```
