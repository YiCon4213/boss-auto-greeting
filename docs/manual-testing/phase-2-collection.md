# Phase 2 浏览器采集人工验证清单

本清单必须在用户本人已登录、可正常手动浏览 BOSS 直聘的浏览器中执行。只验证读取和采集，不进入聊天、不发送消息；遇到验证码、登录失效、安全校验或访问阻断时立即停止，不尝试处理或绕过。

## 前置准备

- [x] 当前分支已迁移到 Alembic `0002 (head)`。
- [x] 全量 pytest、Userscript 契约与 `node --check zhipin-auto-greeting.user.js` 已通过。
- [x] 本地服务仅监听 `127.0.0.1:8765`。
- [x] 已运行 `python -m agent_app.cli show-browser-token`，并由用户手工把浏览器令牌填入油猴面板。
- [x] BOSS 岗位列表页由用户本人登录并可正常手动浏览。

## 两岗位采集

1. 启动本地服务，确认 `GET /api/health` 返回 200。
2. 重新加载已安装的 `zhipin-auto-greeting.user.js`，打开 BOSS 岗位列表。
3. 在油猴面板启用“本地 Agent”，服务地址保持 `http://127.0.0.1:8765`，点击“检查连接”。
4. 通过应用 API 创建 `limit=2` 的批次，再调用 `POST /api/v1/batches/{id}/collect`。
5. 观察浏览器只依次选择两张岗位卡片并读取右侧详情；不得打开聊天页，不得点击“立即沟通/继续沟通”，不得发送消息。
6. 确认批次最终为 `collected`，计数为 2；SQLite 中存在两个不同强身份键的不可变 `job_snapshots`。
7. 重复提交同一岗位或刷新页面后恢复任务，确认快照数不增加、原 payload 和 JD 指纹不变。

## 安全暂停

- [x] 成功重跑期间未出现验证码、登录失效或安全校验。此前曾在任务创建前遇到 BOSS 安全页，当时立即停止且未点击验证；自动化测试覆盖 `paused_security`、批次暂停和已有快照保留，不以绕过验证换取人工证据。

## 独立模式回归

1. 关闭“本地 Agent”开关并停止本地服务。
2. 重新加载岗位列表和油猴脚本。
3. 确认原面板、原配置和“启动”按钮仍可用。
4. 点击“启动”后确认进入原岗位列表循环；在实际发送前人工停止，避免产生未授权沟通。
5. 确认关闭 Agent 模式时不依赖 `127.0.0.1:8765`。
