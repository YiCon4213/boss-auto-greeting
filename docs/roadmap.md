# 项目改造路线图

本路线图用于划分实施顺序和阶段退出条件。详细计划已经生成，当前事实以本路线图、计划复选框和 `docs/current-state-analysis.md` 为准。

## 阶段状态

| 阶段 | 状态 | 说明 |
| --- | --- | --- |
| Phase 0 | 自动护栏完成，人工待验证 | 静态语法、独立模式契约和人工清单已具备；真实独立模式回归并入 Phase 2 退出门禁 |
| Phase 1 | 已完成 | 16 项测试通过，`agent_app` 覆盖率 93%，已合入 `main` |
| Phase 2 | 实现完成，退出门禁待验证 | 45 项测试、93% 覆盖率、Alembic `0002` 与 Userscript 语法通过；两岗位人工验证待执行 |
| Phase 3 | 等待 Phase 2 退出门禁 | 不得在真实浏览器验证通过前开始分析、话术和审批工作台 |
| Phase 4 | 等待 Phase 3 | 已批准队列安全执行、命令和外部 API |

Phase 0 的自动护栏已经随 Phase 2 补齐；真实独立模式回归仍是 Phase 2 退出门禁的一部分，不允许用静态契约替代。

## Phase 0：建立安全基线和测试护栏

目标：在增加 Agent 功能前，锁定原油猴独立模式的关键行为。

产物：

- JavaScript 静态检查和最小测试运行器。
- 现有配置归一化、岗位身份、模板插值和状态转换的契约测试。
- 独立模式手工回归清单。
- 新运行时数据目录和秘密文件的 `.gitignore` 规则。

退出条件：

- 原脚本可直接安装。
- Agent 相关配置默认关闭。
- 独立模式关键契约具有可重复验证方式。

## Phase 1：本地服务、SQLite 和版本化 API 骨架

目标：建立可扩展但最小的本地应用边界。

产物：

- FastAPI 应用和本地启动方式。
- SQLite 模型、迁移和仓储层。
- `/api/v1` 健康检查、画像、设置和批次基础接口。
- 本机令牌、CORS 和密钥存储抽象。
- API/数据库测试。

退出条件：

- 服务只监听本机。
- 数据可跨重启保存。
- API Key 不出现在数据库普通表和接口响应中。

## Phase 2：油猴 AgentBridge 与批次采集

目标：在不破坏独立模式的前提下，让后端请求油猴采集职位快照。

产物：

- Agent 服务在线检测和模式开关。
- 浏览器任务轮询、租约、确认、进度和结果协议。
- `BatchCollector` 只采集不发送。
- 职位强 ID、JD 指纹和快照持久化。
- 采集 10 条后停止并回报。

退出条件：

- Agent 未启动时独立模式完全可用。
- Agent 模式采集期间不会进入聊天或发送。
- 页面刷新和重复任务不会产生重复快照。

## Phase 3：画像、模型分析和个性化话术

目标：完成结构化分析与风格约束生成。

产物：

- 画像模板、字段可见性和基础话术管理。
- OpenAI 兼容模型适配器。
- 求职目标/个人匹配双评分 schema。
- 默认 30 分宽松筛选。
- 轻量个性化话术生成与防虚构校验。
- 模型失败和有限重试处理。

退出条件：

- 空字段不影响分数。
- 个人匹配分不会淘汰岗位。
- 敏感字段默认不进入模型请求。
- 模型失败的岗位不会进入发送队列。

## Phase 4：简约工作台与人工审批

目标：提供可用的画像、设置、批次和分栏审批 UI。

产物：

- 简单页面导航。
- 当前批次进度。
- 左侧岗位队列、右侧分析和话术编辑。
- 取消/恢复、重新生成、保存人工终稿。
- 批次批准和不可变审批版本。

退出条件：

- 低分岗位可恢复。
- 未批准批次不能执行。
- 人工终稿不会被模型或执行过程覆盖。

## Phase 5：安全发送队列与刷新恢复

目标：在列表刷新和重排后安全执行获批队列。

产物：

- `ApprovedQueueRunner`。
- 求职期望恢复和强 ID 重新定位。
- 岗位、详情和会话身份校验。
- 随机间隔、单项失败继续和安全类暂停。
- 批次报告和失败项重新执行。

退出条件：

- 不能通过列表序号发送。
- 身份不一致绝不发送。
- JD 更新但强 ID 相同时按已确认规则继续。
- 单项失败不阻塞后续岗位。
- 验证码和登录失效会暂停整批。

## Phase 6：自然语言 Agent 与外部子 Agent 契约

目标：提供简单自然语言入口，并验证其它项目可以安全调用。

产物：

- 白名单自然语言命令 schema。
- 对话页及任务结果跳转。
- 外部调用文档和示例客户端。
- 权限范围与审批门禁测试。

退出条件：

- 自然语言不能生成任意执行代码。
- “发送”命令不能绕过审批。
- 外部项目能创建、查询批次并取得审批 URL。

## Phase 7：端到端验证与交付

目标：验证完整首版并完成用户文档。

产物：

- 全量自动化测试报告。
- 本人登录环境下的小批量人工验证记录。
- 安装、启动、画像、模型、审批、恢复和清理说明。
- 隐私、限制和故障排查说明。

退出条件：

- 产品需求中的十项验收标准全部通过。
- 独立模式无回归。
- 用户能够从新安装完成一个安全审批批次。
