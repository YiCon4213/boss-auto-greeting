# 当前项目现状审计

## 1. 仓库边界

当前目录是一个独立 Git 仓库，位于父项目 `D:\Web\boss` 内部。父项目是另一个 Git 仓库。后续工作必须只发生在本仓库中，不能把父项目文件作为待修改文件。

本仓库当前核心文件：

- `zhipin-auto-greeting.user.js`：主油猴脚本。
- `zhipin-devtools-unlock.user.js`：本地调试辅助脚本。
- `README.md`：简要说明。
- `AGENTS.md`：维护约束。

当前没有包管理器、构建流程、自动化测试、本地后端或数据库迁移系统。

## 2. 技术栈

- 原生 JavaScript ES6+。
- Tampermonkey/油猴 Userscript。
- Chrome/Edge 浏览器页面环境。
- DOM、History API、MutationObserver、fetch、XMLHttpRequest、PerformanceObserver。
- IndexedDB 保存岗位记录，localStorage 保存配置、运行状态和调试事件。
- `GM_xmlhttpRequest` 用于外部接口和网络兜底。
- SheetJS 通过 CDN 按需加载并导出 Excel。

## 3. 已有能力

### 3.1 页面与网络采集

主脚本在 `document-start` 注入，覆盖职位列表页和聊天页。它会：

- 拦截岗位列表及详情 API。
- 在拦截不可靠时通过 Resource Timing 记录真实请求。
- 主动重放可重放的列表请求。
- 主动补取岗位详情 API。
- 抓取岗位详情 HTML 和公司页面 HTML 补充字段。
- 从 DOM、列表接口、详情接口和 HTML 多路合并职位数据。

### 3.2 职位数据

现有归一化模型已经包含：

- 岗位名称、薪资、公司、城市。
- 工作经验、学历、技能标签、职位描述、工作地址。
- Boss 姓名、职位、活跃状态和在线状态。
- 公司规模、行业、融资阶段、福利、简介。
- 部分工商信息。
- `encryptJobId`、`securityId`、`lid` 等岗位身份字段。

### 3.3 自动沟通

现有状态机是 `list -> chat -> returning -> list`：

- 遍历虚拟岗位列表。
- 通过强标识、完整签名和弱签名匹配卡片。
- 支持公司筛选、公司黑名单、Boss 活跃度筛选、已沟通判重。
- 点击岗位后等待对应详情和沟通按钮稳定。
- 进入聊天页，写入问候语并验证消息是否真正出现。
- 返回列表并尝试恢复原 URL、求职期望、滚动位置和扫描进度。
- 支持随机间隔、最大沟通数、聊天打开重试和异常暂停。

### 3.4 问候语与外部接口

现有脚本支持：

- BOSS 常用语。
- 固定自定义文本和字段插值。
- 调用任意外部接口返回文本。

外部接口能力已经提供模型接入原型，但当前没有：

- 求职目标画像。
- 个人画像字段权限。
- 结构化匹配分析。
- 宽松筛选阈值。
- 批次审批。
- 模型输出契约校验。
- API Key 安全存储。

### 3.5 存储和导出

IndexedDB 当前保存 `jobRecords`、`greetedJobs` 和 `settings`。支持 JSON/Excel 导出、全部清理、按时间清理和已沟通索引重建。

## 4. 主要工程风险

1. 主脚本接近九千行，采集、状态机、UI、存储、网络和发送高度集中。
2. 没有自动化测试，DOM 选择器和内部 API 变化只能依赖人工验证。
3. 继续把模型密钥、画像、审批和 Agent 对话放进油猴脚本，会扩大敏感信息暴露和维护风险。
4. 当前状态机适合实时逐条沟通，不足以直接保证“先采集、延迟审批、再发送”的快照一致性。
5. 列表返回后可能刷新、重排或切换到推荐职位；不能用卡片序号或旧 DOM 引用发送。
6. 调试解锁脚本会改写多种浏览器原生能力，不应成为生产依赖。

## 5. 可复用边界

应复用：

- 列表/详情网络拦截与数据归一化。
- DOM 卡片与接口岗位的身份匹配。
- 求职期望恢复、列表扫描和强 ID 判重。
- 聊天页输入、发送和发送结果验证。
- 独立模式现有功能。

应新增到本地 Agent：

- 自然语言任务入口。
- 画像、模型配置和密钥管理。
- SQLite 职位快照与批次数据。
- 匹配分析和话术生成。
- 审批门禁和不可变发送队列。
- 稳定、版本化、可供外部项目调用的 API。

应逐步从油猴脚本中隔离但不要求首版重写：

- AgentBridge：本机服务检测、任务轮询和结果回报。
- BatchCollector：只采集、不发送的批次采集器。
- ApprovedQueueRunner：只执行已批准队列的发送器。
- StandaloneAutomation：现有独立自动沟通状态机。

## 6. 已验证状态

- 两个 Userscript 均通过 JavaScript 语法检查。
- 新仓库没有现成自动化测试。
- 最近提交集中在列表返回、求职期望恢复和 MutationObserver 等稳定性修复，说明列表状态恢复是当前代码的高风险区域。
